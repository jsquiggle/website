How it works

Opinionated defaults, no black boxes. Bring your own Git and ticketing, or use ours.

  • 1
    Connect
    Securely connect via API or SSH to your device inventory from a jump host or container.
  • 2
    Scan
    Collectors run checks (CIS/NIST/Zero Trust) and compare against desired state per site/role.
  • 3
    Fix
    Generate change‑safe scripts or PRs. Gate applies behind approvals/windows.
  • 4
    Report
    Push results to email, Slack/Teams, SIEM, or a reporting API for auditors.
Sample policy YAML
policy: cis_iosxe_baseline
version: 1.2.0
scope:
  vendors: [cisco, meraki, paloalto, fortinet]
  roles: [core, distribution, access, firewall]
checks:
  - id: no_telnet
    title: Disable Telnet
    when: vendor in [cisco, aruba]
    query: |
      show running-config | include line vty
    assert: not includes('transport input telnet')
    fix: |
      line vty 0 4
      transport input ssh
  - id: radius_timeout
    title: NPS Timeout >= 5s
    when: vendor == cisco
    query: |
      show run | sec radius-server
    assert: includes('timeout 5')
    fix: radius-server timeout 5

FAQ

Do you run in our environment or yours?
You choose. Most customers run our collectors in their network (container/VM) and keep data in their systems. We also offer a managed option for smaller teams.
How do fixes apply safely?
We generate change scripts or Git PRs for your review. You can require approvals and windowing before any apply step.
What if we don’t use Git yet?
No problem. We’ll deliver human‑readable remediation scripts (Ansible/Netmiko/PowerShell) with diff previews and rollbacks.
Which frameworks are supported?
CIS Benchmarks, NIST 800‑53 controls mapping, and Zero Trust baseline packs. Enterprise adds custom control libraries.
